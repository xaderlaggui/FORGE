ForgeTheme Design System

Comprehensive Color Scheme & UI Token Documentation

Design Philosophy

ForgeTheme is a high-fidelity, adaptive user experience system. Built on a rich graphite layout in dark mode and an editorial off-white in light mode, the layout leverages highly vibrant accents to categorize active parameters of training, hydration, nutrition, and intelligent AI interaction.

🎨 Premium Aesthetic Rule
Never hardcode color strings in components (e.g. #FFFFFF or #FF5C2E). Always bind styles to the reactive T.colors token registry, which automatically adapts colors when the theme shifts.

1. Brand & Semantic Accent Palette

These colors remain consistent across both Light and Dark modes to anchor the user interface's visual consistency.

forge #FF5C2E Signature Brand Orange. Buttons & active nav.
green #32D74B Protein tracking & AI sparkles indicator.
blue #0A84FF Carbohydrates & Hydration water graphs.
gold #FF9F0A Fats tracking & workout streak metrics.
red #FF453A Swipe deletion & calorie limit warnings.
purple #BF5AF2 AI Personal Coach and Physique metrics.

2. Adaptive Interface Bases

Surface, border, and typography tokens dynamically shift according to the system scheme setting.

Token Name Dark Mode Value Light Mode Value Layout Usage Context
bg0 #111113 #F5F5F7 Main viewport scaffold and frame page background.
bg1 #1C1C1E #FFFFFF Standard block card component, log module surface.
bg2 #2C2C2E #FBFBFD Subtle nested headers, secondary tables or item rows.
bg3 #3A3A3C #E5E5EA Input bar backgrounds, macro sliders track limits.
t1 #F5F5F7 #1D1D1F Primary text titles, bold units and core logs stats labels.
t2 #AEAEB2 #6E6E73 Secondary caption summaries, macro values, card metadata.
b1 #2C2C2E #D2D2D7 Subtle separation lines and cards outer perimeters borders.

3. Dynamic Feature Mapping

To retain semantic weight across screens, components map to exact color indices in the accent spectrum:

  • Forge Accent (Orange): Main action tabs, streak numbers, and highlight active components.
  • AI Parse & Sparkles (Green): AI-powered sparkles next to meal log lists to denote items logged from generative parsing inputs.
  • Hydration Flow (Blue): Hydration level bar fill metrics, droplet log triggers, glass and bottle actions.
  • Action Triggers (Red): Behind-the-card swipe actions that initiate delete queries.

4. Implementation Pattern

Retrieve the theme objects reactively in custom screens using standard syntax:

import React from 'react';
import { View, Text, StyleSheet } from 'react-native';
import { useForgeTheme } from "@/hooks/useForgeTheme";

export function ActiveMetricCard() {
  const { T } = useForgeTheme();
  const styles = useStyles(T);

  return (
    <View style={styles.card}>
      <Text style={styles.text}>Reactive Layout Token</Text>
    </View>
  );
}

const useStyles = (T: any) => StyleSheet.create({
  card: {
    backgroundColor: T.colors.bg1,
    padding: T.spacing.px4,
    borderWidth: 1,
    borderColor: T.colors.b1,
    ...T.shadows.lift,
  },
  text: {
    color: T.colors.t1,
    fontSize: T.typography.sizes.body,
  }
});
